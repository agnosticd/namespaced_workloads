= ocp4_workload_tenant_gitea_user - Create a Gitea tenant user and migrate repositories

== Role overview

* This role creates a user on an existing Gitea instance deployed via the Gitea operator.
* Optionally migrates a list of repositories from external sources to the user's account.
* Supports teardown by deleting all user repositories and the user account.

== Requirements

* A Gitea instance deployed via the Gitea operator (CR kind: `Gitea`, apiVersion: `pfe.rhpds.com/v1`).
* The Gitea admin setup must be complete (`status.adminSetupComplete: true`).

Example deployment for the Shared Cluster:

[source,yaml]
----
ocp4_workload_gitea_operator_project: gitea
ocp4_workload_gitea_operator_catalog_image_tag: v2.1.0
ocp4_workload_gitea_operator_catalog_image: quay.io/rhpds/gitea-catalog

ocp4_workload_gitea_operator_deploy_gitea_instance: true
ocp4_workload_gitea_operator_gitea_image: quay.io/rhpds/gitea
ocp4_workload_gitea_operator_gitea_image_tag: "1.25.5"
ocp4_workload_gitea_operator_name: gitea
ocp4_workload_gitea_operator_gitea_hostname: gitea
ocp4_workload_gitea_operator_gitea_volume_size: 5Gi
# ocp4_workload_gitea_operator_gitea_volume_storage_class: ocs-external-storagecluster-cephfs
ocp4_workload_gitea_operator_postgresql_image: registry.redhat.io/rhel10/postgresql-16
ocp4_workload_gitea_operator_postgresql_image_tag: "10.1"
ocp4_workload_gitea_operator_postgresql_volume_size: 5Gi
# ocp4_workload_gitea_operator_postgresql_volume_storage_class: ocs-external-storagecluster-cephfs
ocp4_workload_gitea_operator_ssl_route: true

ocp4_workload_gitea_operator_create_admin: true
ocp4_workload_gitea_operator_admin_user: gitea-admin
ocp4_workload_gitea_operator_admin_email: gitea-admin@redhat.com
ocp4_workload_gitea_operator_admin_password: "{{ common_admin_password}}"

ocp4_workload_gitea_operator_create_users: false
----
